# 自动化公众号内容生产线蓝图

目标不是再做一个排版器，而是做一条“知识来源 -> 选题 -> 成稿 -> 排版 -> 草稿箱 -> 手机确认发布 -> 线索承接”的内容生产线。

## 目标工作流

1. 每天同步 Obsidian 笔记、飞书云文档、收藏链接和素材库。
2. 抓取当天 AI 产品、AI 科技、企业知识库、RAG、Agent、SaaS 获客相关新动态。
3. 根据主线业务“AI 企业知识库服务”生成候选选题。
4. 选择 1 个最适合传播和获客的选题，生成文章大纲。
5. 从个人知识库和最新素材中引用证据，写成公众号文章。
6. 自动排版成微信兼容富文本。
7. 通过微信公众号接口创建草稿。
8. 给作者手机发送预览和确认卡片。
9. 作者确认后提交发布或预约发布。
10. 发布后记录阅读、分享、线索和私域转化数据，反哺下一次选题。

## 系统模块

### 1. Knowledge Ingestor

负责把每天的输入变成可检索素材。

- Obsidian：读取本地 vault 的 Markdown 文件，按修改时间、标签、双链、标题层级抽取知识块。
- 飞书云文档：通过飞书开放平台读取指定知识库、文档、表格或文件夹。
- 网页/视频素材：保存标题、URL、摘要、发布时间、来源可信度。
- 手动输入：保留一个“今日重点”入口，让作者手动指定想表达的观点。

输出统一为：

```json
{
  "id": "source-id",
  "source": "obsidian|lark|web|youtube|manual",
  "title": "素材标题",
  "content": "正文或摘要",
  "url": "可选来源链接",
  "tags": ["AI企业知识库", "获客"],
  "created_at": "2026-05-27T09:00:00+08:00",
  "importance": 0.82
}
```

### 2. Trend Collector

负责每天抓取外部新信息。

- Google News / RSS：抓 AI 产品、企业 AI、知识库、RAG、Agent、SaaS。
- YouTube：抓指定频道最新视频和标题摘要。
- X / Twitter：建议通过自建 RSS Bridge 或第三方合法 API，不建议硬爬。
- GitHub Trending / Product Hunt / Hacker News：补充开发者和产品侧信号。

### 3. Topic Strategist

从“个人知识资产 + 最新趋势 + 主线业务”里选择题目。

评分维度：

- 对目标客户是否有用：企业老板、销售负责人、交付负责人、知识管理负责人。
- 是否能体现你的专业服务：AI 企业知识库诊断、搭建、培训、落地。
- 是否有传播钩子：新产品、新方法、反常识、可复制清单。
- 是否有素材支撑：至少 3 个可靠来源 + 1 个个人观点。
- 是否能自然引导咨询：文章结尾能提出具体场景问题。

### 4. Writer Agent

建议固定文章结构，减少每天随机发挥：

```text
标题：一句明确承诺或反常识观点
开头：今天发生了什么 / 为什么和读者有关
核心判断：一句话结论
正文一：现象与案例
正文二：对企业知识库/获客/交付的启发
正文三：可执行清单
结尾：提问 + 轻 CTA
```

### 5. Layout Renderer

把文章转换为微信公众号可粘贴 HTML。

- 所有样式内联。
- 图片必须是稳定可访问 URL，不能依赖本地路径。
- 代码、引用、表格、卡片都要做微信兼容降级。
- 输出同时保留 Markdown、HTML、JSON 元数据。

### 6. WeChat Publisher

浏览器前端不能保存公众号 appsecret，因此必须放在后端。

推荐后端流程：

1. 用公众号 `appID` 和 `appsecret` 换取 `access_token`。
2. 上传图文内图片素材，替换文章里的图片 URL。
3. 调用草稿箱接口创建草稿。
4. 返回草稿 ID 和预览信息。
5. 作者在手机上确认。
6. 调用发布接口提交发布，或只停在草稿箱等待人工发布。

安全建议：

- appsecret 只放服务端密钥管理，不进浏览器、不进 GitHub。
- 第一期最好只做到“创建草稿 + 手机确认”，发布按钮由人按。
- 如果要自动提交发布，必须保留人工确认日志。

### 7. Mobile Approval

作者只做三件事：

- 看标题、封面、摘要和正文预览。
- 选择：通过 / 退回修改 / 改标题。
- 通过后创建草稿或提交发布。

可以通过飞书机器人、企业微信机器人、邮件或一个手机网页完成确认。

## MVP 版本拆分

### V1：当前开源排版器增强

- 粘贴整篇文章后自动结构化排版。
- 本地预览和富文本复制。
- 发布前检查。

### V2：半自动草稿箱

- 增加后端服务。
- 配置公众号 appID/appsecret。
- 支持图片上传到公众号素材。
- 一键创建公众号草稿。
- 手机端确认后再进入草稿箱。

### V3：知识源自动写作

- Obsidian vault 索引。
- 飞书云文档同步。
- RSS/YouTube/GitHub/Product Hunt 趋势抓取。
- 每天生成 3 个选题候选。
- 人工选题后自动成稿。

### V4：获客闭环

- 文章结尾 CTA 模板。
- 线索表单或私域入口。
- 阅读和转化数据回收。
- 根据转化表现优化下一周选题。

## 推荐技术栈

- Frontend：当前静态工具继续演进，后续可迁移到 Next.js。
- Backend：Python FastAPI 或 Node.js Hono/Express。
- Queue：本地 cron 起步，后续用 GitHub Actions、Vercel Cron 或云函数定时。
- Database：SQLite 起步，后续 PostgreSQL。
- Vector Search：SQLite FTS 起步，后续 pgvector 或 Qdrant。
- LLM：用于选题、摘要、写作、改标题、检查事实引用。

## 最小服务端接口

```http
POST /api/sources/sync
POST /api/topics/generate
POST /api/articles/draft
POST /api/articles/render
POST /api/wechat/drafts
POST /api/approvals/send
POST /api/approvals/:id/confirm
```

## 每天自动化脚本

```text
08:00 同步 Obsidian 和飞书
08:10 抓取 AI 趋势和竞品动态
08:20 生成候选选题
08:30 发给作者手机选择
09:00 生成初稿
09:10 自动排版并创建预览
09:20 作者手机确认
09:30 创建公众号草稿或提交发布
```

## 不建议一开始做的事

- 不建议一开始全自动发布。公众号内容牵涉品牌和合规，先保留人工确认。
- 不建议浏览器模拟点击公众号后台。稳定性和安全性都不如官方接口。
- 不建议把所有素材都塞给模型。先检索出相关来源，再让模型写。
- 不建议只追热点。每篇都要和“AI 企业知识库服务”建立业务相关性。
