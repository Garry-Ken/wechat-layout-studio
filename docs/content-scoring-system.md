# 内容评分与校准系统

本方案参考 `XBuilderLAB/cheat-on-content`，但不照搬它的观点视频 rubric。它真正值得吸收的是方法论：

```text
打分 -> 盲预测 -> 发布 -> T+N 复盘 -> 升级 rubric
```

对公众号知识库写作来说，评分不是为了“判断这篇像不像爆款”，而是为了让 Agent 在写作前、中、后都能回答：

- 这篇是否服务主题？
- 是否真的调用了你的知识库，而不是写泛泛 AI 文？
- 是否有信息增量？
- 是否能被目标客户转发、收藏、咨询？
- 哪些判断被发布后的数据验证或推翻？

## 从 cheat-on-content 借鉴的机制

### 1. 预测前评分

文章发布前必须留下结构化判断，而不是发布后再解释。

要记录：

- 维度分
- 综合分
- 预测阅读/分享/收藏/咨询区间
- 一句话预测理由
- 关键假设

### 2. 盲预测

发布后数据出来之前写预测。看到阅读、点赞、评论、分享之后，不能回头改预测，只能追加复盘。

这能避免“事后诸葛亮”。

### 3. 复盘闭环

公众号建议默认 T+7d 复盘，因为长文传播和私域转发比短视频慢。

复盘数据：

- 阅读量
- 分享数
- 收藏数
- 点赞/在看
- 评论关键词
- 私信/咨询/加微信线索
- 文章带来的业务对话数量

### 4. Rubric 可升级

评分维度不是一次定死的。每次复盘后，如果出现系统性误判，就升级权重或维度。

例如：

- 连续 3 篇高估阅读，可能标题钩子权重太高。
- 连续 3 篇收藏高但咨询低，可能实用性高但商业承接弱。
- 一篇阅读一般但咨询很多，说明“获客价值”维度应独立于传播分。

## 公众号知识库文章 Rubric v0

v0 是 cold-start 占位版，等权起步。前 5-10 篇主要用于收集校准数据，不要过度相信综合分。

```text
composite = (TS + KA + IG + UV + SC + CV + SN + CE) / 8 × 2.0
```

每个维度 0-5 分，综合分范围 0-10。

## 8 个维度

### TS — Topic Strength（主题势能）

这个主题是否值得现在写？

- 0：过时、泛泛、没有明确读者
- 3：和近期趋势有关，但角度普通
- 5：同时连接热点、痛点和你的业务主线

典型高分信号：

- 正好有新产品、新政策、新行业变化
- 目标客户最近正在遇到
- 能自然连接 AI 企业知识库服务

### KA — Knowledge Alignment（知识库适配度）

这篇是否真正调用了你的 Obsidian/飞书知识，而不是模型空写？

- 0：几乎没有个人知识资产
- 3：引用了几条笔记或素材
- 5：多个内部经验、方法论、案例被重新组合成新观点

典型高分信号：

- 使用了客户会议纪要、销售话术、交付 SOP、读书笔记
- 有“只有你会这么讲”的经验判断
- 不是外部热点摘要

### IG — Information Gain（信息增量）

读者看完是否获得了新知识、新框架或新判断？

- 0：常识复述
- 3：有一两个有用观点
- 5：能让读者重构一个问题的理解

典型高分信号：

- 命名了读者知道但说不清的问题
- 给出新分类、新框架、新路径
- 不是“AI 很重要”这种空话

### UV — User Value（读者可用性）

读者是否能拿去行动？

- 0：看完不知道干什么
- 3：有建议但不够具体
- 5：有清单、步骤、判断标准或模板

典型高分信号：

- “今天就能做的 3 件事”
- “判断企业知识库是否值得做的 5 个问题”
- “销售团队先从客户问题库开始”

### SC — Shareability（传播/转发冲动）

读者会不会愿意转发给同事、老板、客户或朋友圈？

- 0：没有转发动机
- 3：有用但不够有表达欲
- 5：转发本身能帮读者表达立场、焦虑、方法或身份

典型高分信号：

- 有反常识标题
- 有可截图金句
- 有“这说的就是我们公司”的识别感
- 转发不会暴露尴尬处境，反而显得专业

### CV — Commercial Value（商业承接）

文章是否自然连接你的主线业务和获客路径？

- 0：和业务无关
- 3：结尾能提到服务，但略硬
- 5：全文问题定义本身就自然导向你的服务

典型高分信号：

- 读者看完会意识到自己需要诊断企业知识库
- 结尾 CTA 是场景提问，不是硬广
- 能引出咨询、诊断、模板领取、私域沟通

### SN — Structure & Narrative（结构叙事）

文章是否像真人有取舍地编排，而不是资料拼贴？

- 0：素材堆砌
- 3：有结构但转折弱
- 5：开头、判断、论证、案例、方法、CTA 连成一条线

典型高分信号：

- 第一屏抓住问题
- 每一节服务核心判断
- 案例不是散点，而是证明链的一部分

### CE — Credibility Evidence（可信证据）

观点是否有足够证据支撑？

- 0：全是主观判断
- 3：有来源或案例，但不完整
- 5：外部趋势 + 内部案例 + 个人经验互相印证

典型高分信号：

- 来自飞书/Obsidian 的具体业务经验
- 外部产品/行业动态有来源
- 对不确定事实明确标注，不装确定

## 双分制：传播分与获客分

公众号文章不能只看阅读量。对“AI 企业知识库服务”这种业务，建议拆成两条分数：

```text
传播分 = (TS + IG + SC + SN + CE) / 5 × 2.0
获客分 = (KA + UV + CV + CE + TS) / 5 × 2.0
```

含义：

- 传播分高：适合发朋友圈、扩大影响、拉新读者。
- 获客分高：适合沉淀案例、促成咨询、发给潜在客户。
- 两者都高：优先级最高。
- 传播高获客低：可以做品牌内容，但 CTA 要轻。
- 获客高传播低：可以做销售材料、私域文章，不一定追求广泛传播。

## 预测日志格式

每篇文章发布前生成一个预测记录：

```markdown
# <文章标题> — 内容预测

**Article ID**: <hash>
**Theme**: <用户输入主题>
**Target Reader**: <目标读者>
**Business Line**: AI 企业知识库服务
**Rubric Version**: v0
**Prediction Time**: 2026-05-27
**Knowledge Sources Used**: Obsidian 12 条 / 飞书 4 条 / 外部趋势 3 条
**Data Status**: blind

## 评分

| 维度 | 分 | 理由 |
|---|---:|---|
| TS | 4 | 主题连接 AI 趋势和企业真实痛点 |
| KA | 5 | 调用了客户问题库、销售话术、交付 SOP |
| IG | 4 | 提出“知识底座先于 Agent”的判断 |
| UV | 4 | 给出客户问题库起步清单 |
| SC | 3 | 有金句，但可转发身份表达还不够强 |
| CV | 5 | 自然导向知识库诊断服务 |
| SN | 4 | 结构完整 |
| CE | 3 | 外部数据来源还可加强 |

传播分：7.2
获客分：8.4
综合分：8.0

## 预测

- 阅读：基础盘 / 命中 / 小爆 / 大爆
- 分享：中
- 收藏：高
- 咨询线索：2-5 个

## 关键假设

如果这篇咨询高但阅读一般，说明 CV/UV 比 SC 更能预测业务价值。
如果阅读高但咨询低，说明主题有传播性但业务承接不够。
```

```markdown
- 点赞/在看：
- 评论关键词：

- 哪些维度被验证？

- 哪些维度被低估？

- 哪些段落被评论/转发引用？

### Rubric 观察

- [ ] 是否需要调整某个维度权重？
- [ ] 是否出现新维度？
- [ ] 是否需要调整 bucket 边界？
```

## 候选主题评分

在真正写文章前，候选主题也要评分，但维度更轻：

```text
candidate_score = TS×1.5 + KA + IG + SC + CV
```

候选主题 schema：

```json
{
  "id": "12-char-hash",
  "theme": "AI 企业知识库为什么会成为中小企业刚需",
  "target_reader": "中小企业老板",
  "source_signals": ["obsidian", "lark", "rss"],
  "dimension_scores": {
    "TS": 5,
    "KA": 4,
    "IG": 4,
    "SC": 3,
    "CV": 5
  },
  "recommended_angle": "不是缺 AI 工具，而是缺让经验复用的知识底座",
  "risk": "如果案例不足，容易写成泛泛趋势文"
}
```

## 如何组合进知识库写作 Agent

### 写作前

1. 根据用户主题生成 3-5 个可写角度。
2. 每个角度跑候选主题评分。
3. 选择传播分和获客分平衡最好的角度。
4. 生成检索计划：需要哪些 Obsidian、飞书、外部趋势素材。

### 检索后

1. 对每条素材打“贡献分”。
2. 排除低相关、低增量、不能支撑核心判断的素材。
3. 保留证据链：外部趋势、内部案例、个人判断、可执行方法。

### 初稿后

1. 跑 8 维文章评分。
2. 如果 SC 低，要求重写标题、开头和金句。
3. 如果 KA 低，回到知识库继续检索。
4. 如果 CV 低，重写业务承接和 CTA。
5. 如果 CE 低，补来源和案例。

### 发布后

1. T+7d 复盘。
2. 把表现和预测比较。
3. 把有效观察写回 rubric。
4. 达到 5/10/20 篇时重新校准权重。

## 初始权重建议

在没有历史数据前，先不要给复杂权重。建议 v0 等权。

当累计 5 篇以上数据后，可以尝试：

```text
composite_v1 = (TS×1.2 + KA×1.2 + IG×1.4 + UV + SC×1.3 + CV×1.5 + SN + CE) / 9.6 × 2.0
```

这只是候选，不应直接升正。必须用历史样本回测。

## 关键原则

- 不复盘的评分没有意义。
- 不看咨询线索的公众号评分是不完整的。
- 不调用私人知识库的文章，即使流量高，也不一定服务你的业务。
- 不要迷信第一版 rubric。真正的公式来自你自己的 10-20 篇数据。
- 评分不是为了束缚写作，而是为了让 Agent 学会你的内容判断。
